Timer/PWM Driver

Overview

The TPM module is a timer that supports input capture, ouput compare and generation of PWM signals. The current SDK driver supports support all the above features.

Initialization

To initialize the TPM driver, call the TPM_DRV_Init() function and pass the instance number of the TPM you want to use. For instance, to use TPM0, pass a value of 0 to the init function. In addition, you should also pass a user configuration structure tpm_general_config_t, as shown here:
// TPM configuration structure for user. User needs to set the relevant ones.
typedef struct TpmGeneralConfig {
bool isDBGMode;
bool isGlobalTimeBase;
bool isTriggerMode;
bool isStopCountOnOveflow;
bool isCountReloadOnTrig;
tpm_trigger_source_t triggerSource;
}tpm_general_config_t;

Generate a PWM signal

TPM driver user calls the TPM_DRV_PwmStart() function to generate a PWM signal. The user will have to use the below structure to configure different parameters related to this PWM signal.
typedef struct TpmPwmParam
{
tpm_pwm_mode_t mode;
tpm_pwm_edge_mode_t edgeMode;
uint32_t uFrequencyHZ;
uint32_t uDutyCyclePercent;
}tpm_pwm_param_t;
The mode options are kTpmEdgeAlignedPWM and kTpmCenterAlignedPWM. For edge mode, the options available are kTpmHighTrue and kTpmLowTrue. The user will have to specify the PWM signal frequency in HZ and the duty cycle percentage (value between 0-100).

Input Capture

TPM driver user calls the TPM_DRV_InputCaptureEnable() function to setup the channel for input capture. The user will have to use the below enum to specify the capture mode to be used.
typedef enum _tpm_input_capture_mode_t
{
kTpmRisingEdge = 1,
kTpmFallingEdge,
kTpmRiseOrFallEdge
}tpm_input_capture_mode_t;

The user can choose to enable channel interrupts by using the intEnable argument of the function.The user can call the TPM_DRV_GetChnVal() function to get read the captured LPTPM counter value.

Output Compare

TPM driver user calls the TPM_DRV_OutputCompareEnable() function to setup the channel for output compare. The user will have to use the below enum to specify the compare mode to be used.
typedef enum _tpm_output_compare_mode_t
{
kTpmOutputNone = 0,
kTpmToggleOutput,
kTpmClearOutput,
kTpmSetOutput,
kTpmHighPulseOutput,
kTpmLowPulseOutput
}tpm_output_compare_mode_t;

The user can choose to enable channel interrupts by using the intEnable argument of the function. The matchVal argument of the function will be written to the CnV register.

Interrupt handler

The TPM driver provides an interrupt handler for the counter overflow and channel interrupts. These handlers clear the status bits.To add more actions to the default handler, add calls to the functions inside the interrupt handlers TPMx_IRQHandler() function, where x=0, 1, 2 depends on the TPM instance the user is working with.

 If Global time base enabled, true to enable, false to disable.

 If Trigger mode enabled, true to enable, false to disable.
 
bool isStopCountOnOveflow
 True to stop counter after overflow, false to continue running.

 True to reload counter on trigger, false means counter is not reloaded.
